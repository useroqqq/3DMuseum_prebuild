# Музей кафедры фотограмметрии МИИГАиК

Интерактивная веб-экскурсия для музея кафедры фотограмметрии МИИГАиК.

Проект представляет собой одностраничное веб-приложение, позволяющее пользователю знакомиться с экспонатами музея через интерактивный план, страницы групп оборудования, отдельные страницы экспонатов, 3D-модели, фотогалерею, аудиогид и подробные Markdown-описания.

Разработанная веб-экскурсия может использоваться как самостоятельное приложение и как программная основа для дальнейшей интеграции в веб-портал «Электронный Музей фотограмметрии МИИГАиК».

## О проекте

Основная цель проекта — обеспечить наглядное и доступное представление фотограмметрического и геодезического оборудования в веб-среде.

Веб-экскурсия позволяет пользователю не только прочитать сведения об экспонатах, но и взаимодействовать с ними: выбирать объекты на интерактивной сцене, рассматривать 3D-модели, изучать фотографии, прослушивать аудиогид и читать подробные описания.

Проект ориентирован на музей кафедры фотограмметрии МИИГАиК, где представлены приборы и материалы, связанные с развитием фотограмметрических и геодезических технологий.

---

## Функциональные возможности

Пользователь может:

- просматривать интерактивный план музея;
- выбирать экспонаты и группы приборов;
- переходить к странице отдельного экспоната;
- просматривать страницу группы экспонатов;
- вращать и приближать 3D-модель прибора;
- просматривать фотографии экспоната;
- открывать фотографии в полноэкранном режиме;
- слушать аудиогид;
- читать подробное описание экспоната;
- переключать светлую и тёмную тему интерфейса.

## Технологический стек

Проект реализован с использованием современного frontend-стека:

- **TypeScript** — строгая типизация данных и компонентов;
- **React 19** — построение пользовательского интерфейса;
- **Vite** — запуск проекта и production-сборка;
- **React Router DOM v7** — маршрутизация без перезагрузки страницы;
- **Tailwind CSS v4** — стилизация, адаптивная верстка, светлая и тёмная тема;
- **Motion** — анимации и плавные переходы;
- **Lucide React** — иконки интерфейса;
- **Three.js** — работа с 3D-графикой в браузере;
- **@react-three/fiber** — интеграция Three.js с React;
- **@react-three/drei** — вспомогательные инструменты для 3D-сцены;
- **React Markdown** — отображение подробных описаний из Markdown-файлов;
- **Blender** — подготовка и оптимизация 3D-моделей.

## Страницы приложения

Приложение состоит из трёх основных типов страниц:

- главная страница веб-экскурсии;
- страница группы экспонатов;
- страница отдельного экспоната.

Навигация между страницами реализована средствами **React Router DOM**, поэтому переходы выполняются без полной перезагрузки страницы.

---

### Главная страница

Главная страница является начальным экраном интерактивной веб-экскурсии. Она выполняет роль интерактивного плана музея кафедры фотограмметрии.

На главной странице пользователь видит схематичное или трёхмерное представление музейного пространства с размещёнными маркерами экспонатов и групп оборудования. Каждый маркер связан с конкретным объектом из файла `src/data/exhibits.ts`.

Пользователь может:

- осматривать сцену;
- вращать и приближать область просмотра;
- выбирать маркеры экспонатов;
- переходить к странице отдельного прибора;
- переходить к странице группы экспонатов;
- переключать светлую и тёмную тему интерфейса.

Главная страница помогает сохранить пространственный контекст музея: пользователь не просто выбирает объект из списка, а видит примерное расположение приборов в виртуальном пространстве.

**Скриншот главной страницы:**  
![Главная страница веб-экскурсии](https://github.com/useroqqq/3DMuseum/blob/main/illustrations/main.png?raw=true)


### Страница группы экспонатов

Страница группы экспонатов используется для отображения нескольких связанных приборов или оборудования одной категории.

Такой формат применяется, когда один маркер на главной сцене соответствует не одному отдельному объекту, а группе приборов. Например, это может быть набор оборудования одного типа, комплекс приборов или несколько экспонатов, расположенных рядом.

На странице группы пользователь видит список или карточки связанных экспонатов. Каждая карточка содержит название прибора и может использоваться для перехода к странице отдельного экспоната.

Пользователь может:

- просматривать состав выбранной группы;
- выбирать конкретный экспонат внутри группы;
- возвращаться к главной странице;
- переходить к связанным объектам;
- использовать светлую или тёмную тему интерфейса.

Страница группы является промежуточным уровнем навигации между общим планом музея и подробной страницей конкретного прибора.

**Скриншот страницы группы экспонатов:**  
![Страница группы экспонатов](https://github.com/useroqqq/3DMuseum/blob/main/illustrations/group.png?raw=true)


### Страница отдельного экспоната

Страница отдельного экспоната является основной страницей для детального изучения выбранного прибора.

Содержание страницы формируется динамически на основе идентификатора экспоната в URL. Приложение получает `id` из маршрута, находит соответствующую запись в `src/data/exhibits.ts` и отображает связанные с экспонатом материалы.

На странице отдельного экспоната размещаются:

- название прибора;
- краткая информация об экспонате;
- интерактивная 3D-модель;
- фотогалерея;
- аудиогид;
- подробное описание в формате Markdown;
- элементы навигации;
- переключатель светлой и тёмной темы.

3D-модель загружается из директории `public/models`, фотографии — из `public/images`, аудиофайл — из `public/audio`, а подробное описание — из `public/descriptions`.

Подробное описание подгружается асинхронно через **Fetch API** и отображается с помощью **React Markdown**. Это позволяет хранить большие текстовые материалы отдельно от основного кода приложения.

Пользователь может:

- вращать и приближать 3D-модель прибора;
- рассматривать прибор с разных сторон;
- переключать фотографии;
- открывать изображение в полноэкранном режиме;
- запускать и останавливать аудиогид;
- читать подробное описание;
- возвращаться к плану музея или к группе экспонатов.

Если какой-либо медиафайл отсутствует, приложение отображает сообщение или заглушку, не прерывая работу страницы.

**Скриншот страницы отдельного экспоната:**  
![Страница отдельного экспоната](https://github.com/useroqqq/3DMuseum/blob/main/illustrations/page.png?raw=true)


### Схема маршрутов приложения

Основные маршруты приложения:

```text
/                  — главная страница интерактивной веб-экскурсии
/group/:id         — страница группы экспонатов
/exhibit/:id       — страница отдельного экспоната
```

Маршруты позволяют сохранять прямые ссылки на отдельные группы и экспонаты. Это удобно для демонстрации, образовательного использования и дальнейшей интеграции веб-экскурсии в электронный музейный портал.

## Основные файлы проекта

### `src/App.tsx`

Главный компонент приложения.

Отвечает за подключение маршрутизации и определение основных страниц:

- главная страница;
- страница группы экспонатов;
- страница отдельного экспоната.

Также в приложении подключается общая структура интерфейса и провайдер темы.

---

### `src/data/exhibits.ts`

Основной модуль данных проекта.

В нём хранятся сведения об экспонатах и группах оборудования.

Для каждого экспоната могут задаваться:

- уникальный идентификатор;
- название;
- краткое описание;
- принадлежность к группе;
- координаты маркера на интерактивной сцене;
- путь к 3D-модели;
- список фотографий;
- путь к аудиофайлу;
- путь к Markdown-описанию;
- признак группы;
- связь с дочерними экспонатами.

Добавление нового экспоната выполняется через добавление новой записи в `exhibits.ts` и размещение соответствующих файлов в директории `public`.

---

### `src/components/MuseumPlan3D.tsx`

Компонент главной интерактивной сцены.

Отвечает за отображение плана музея, расстановку маркеров экспонатов и переход к страницам выбранных объектов.

---

### `src/pages/Exhibit.tsx`

Страница отдельного экспоната.

На основе идентификатора из URL получает данные об экспонате и отображает:

- название;
- аудиоплеер;
- 3D-модель;
- фотогалерею;
- подробное Markdown-описание.

Подробное описание загружается асинхронно из папки `public/descriptions`.

---

### `src/pages/GroupSelection.tsx`

Страница группы экспонатов.

Используется для отображения нескольких связанных приборов или оборудования одной категории.

---

### `src/components/ModelViewer.tsx`

Компонент интерактивного просмотра 3D-моделей.

Использует Three.js, React Three Fiber и Drei.

Поддерживает:

- загрузку `.glb` / `.gltf`-моделей;
- вращение модели;
- приближение и отдаление;
- управление камерой;
- переиспользование загруженных моделей через кеш загрузчика.

---

### `src/components/AudioPlayer.tsx`

Компонент аудиогида.

Использует стандартные возможности `HTMLAudioElement` и React для управления воспроизведением аудиофайлов.

---

### `src/components/ThemeProvider.tsx`

Компонент управления светлой и тёмной темой.

Сохраняет выбранную тему в `localStorage`, поэтому после обновления страницы пользователь возвращается к выбранному оформлению.

---

### `src/components/ThemeToggle.tsx`

Кнопка переключения темы.

Использует состояние из `ThemeProvider` и меняет режим интерфейса.

---

### `src/index.css`

Глобальный файл стилей.

Подключает Tailwind CSS, Typography-плагин для Markdown-текстов и вариант `dark` для тёмной темы.

## Добавление нового экспоната

Чтобы добавить новый экспонат, необходимо выполнить следующие шаги.

### 1. Подготовить 3D-модель

Создать или обработать модель в Blender и экспортировать её в формат `.glb` или `.gltf`.

Файл модели поместить в:

```text
public/models/
```

### 2. Добавить изображения

Фотографии экспоната поместить в:

```text
public/images/
```

### 3. Добавить аудиофайл

Аудиофайл в формате `.mp3` поместить в:

```text
public/audio/
```

### 4. Добавить подробное описание

Создать Markdown-файл с описанием экспоната и поместить его в:

```text
public/descriptions/
```

### 5. Добавить запись в `exhibits.ts`

В файле:

```text
src/data/exhibits.ts
```

добавить объект с данными нового экспоната.

Пример структуры:

```ts
{
  id: "example",
  title: "Название экспоната",
  shortDescription: "Краткое описание экспоната",
  modelPath: "/models/example.glb",
  images: [
    "/images/example_1.jpg",
    "/images/example_2.jpg"
  ],
  audioPath: "/audio/example.mp3",
  descriptionFile: "/descriptions/example.md",
  position3D: [0, 0, 0],
  groupId: "group-example"
}
```

После этого экспонат может быть отображён в интерфейсе приложения.

## Работа с 3D-моделями

3D-модели создаются и дорабатываются в Blender.

В проекте используются разные способы подготовки моделей:

- создание модели вручную с нуля;
- использование LiDAR-скана как основы;
- совмещение LiDAR-скана с вручную созданными элементами;
- минимальная обработка качественного скана.

Перед подключением к приложению модель должна быть:

- очищена от лишней геометрии;
- отцентрирована;
- приведена к корректному масштабу;
- оптимизирована для отображения в браузере;
- экспортирована в `.glb` или `.gltf`.

## Светлая и тёмная тема

В проекте реализовано переключение светлой и тёмной темы.

Механизм работы:

1. пользователь нажимает кнопку переключения темы;
2. состояние темы изменяется в React;
3. на корневой элемент добавляется или удаляется класс `dark`;
4. Tailwind CSS применяет соответствующие классы тёмной темы;
5. выбранная тема сохраняется в `localStorage`.

После обновления страницы приложение считывает сохранённое значение и восстанавливает выбранную тему.

---

## Обработка ошибок

В проекте предусмотрена обработка ситуаций, когда отдельные ресурсы отсутствуют.

Если не найдено изображение, аудио, 3D-модель или Markdown-описание, приложение не должно завершать работу с ошибкой. Вместо этого отображается сообщение или визуальная заглушка.

Это важно, потому что наполнение музейной коллекции может происходить постепенно.

Разработанная веб-экскурсия предназначена для цифрового представления музейных материалов кафедры фотограмметрии, повышения их доступности и наглядности, а также для дальнейшего использования в составе электронного музейного портала.
